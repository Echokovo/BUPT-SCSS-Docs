# 1）需要哪些外模式？它们分别需要哪些信息？

**学生信息管理系统**

1. 学生视图：个人信息、课程名单
2. 教师视图：所教授课程名单与信息、所教授课程学生名单
3. 管理员视图：课程管理、教师管理、学生管理

**火车票订票系统**
1. 用户视图：列车时刻、票价、余票
2. 管理员视图：票价、余票、列车信息、订单信息

**其他系统**
1. 用户视图：用户信息
2. 管理员视图：用户信息、其他信息（订单、商品）

# 2）模式应包含哪些表？

**学生信息管理系统**
1. 学生表：学生姓名、学生班级
2. 课程表：课程名、任课老师名
3. 成绩表：学生姓名、课程名、成绩
4. 教师表：教师姓名、教师联系方式

**火车票订票系统**
1. 列车表：列车编号、始发站、终点站
2. 车次时间表：车次编号、列车编号、时间
3. 订单表：订单ID、用户ID、车次编号
4. 用户表：用户ID、联系方式

**其他系统**
1. 用户表：用户ID、联系方式
2. 订单表：订单ID、用户ID、商品ID
3. 商品表：商品ID

# 3）举例说明怎样保证数据的独立性

**物理数据独立性**

在学生信息管理系统中，当学生数量增大到一定程度后，改用更加高效的存储结构，此时管理员只需要修改模式/内模式映像，使模式保持不变，即可保证物理数据独立性。

**逻辑数据独立性**

在火车票订票系统中，当需要记录列车的具体型号时，只需要在列车表中新增一列，此时管理员只需要修改外模式/模式映像，使得用户视图中能够显示列车型号，使外模式保持不变，即可保证逻辑数据独立性。
